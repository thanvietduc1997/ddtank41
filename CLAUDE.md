# DDTank41 — AI Agent Instructions

## File placement convention

**Do not add new files into the original codebase directories** (`Game.Logic`, `Tank.Data`, `Database`, etc.).  
All files generated by AI agents (scripts, configs, tools, notes) must go in the `ducthan/` directory at the project root.

```
DDTank41/
├── ducthan/          ← all AI-generated files go here
│   ├── restore.sh
│   └── restore.sql
├── Database/         ← original codebase, do not add files here
├── Game.Logic/
└── ...
```

This keeps the original codebase clean and makes it easy to identify what was added versus what was part of the original project.

## Infrastructure

- SQL Server runs in Docker container `ddtank-mssql` on `localhost:1433`
- SA password: `DDTank41@Strong`
- Restore script: `ducthan/restore.sh` (pulls image, starts container, restores all 3 databases)
- Databases: `Player34`, `Game34`, `Db_Membership`

Connect:
```bash
docker exec -it ddtank-mssql /opt/mssql-tools18/bin/sqlcmd \
  -S localhost -U SA -P 'DDTank41@Strong' -No
```
